<objective>
Fix the backward-compatibility break in AppliedFanEntry deserialization and suppress dead_code warnings on test-only daemon functions.

Purpose: Phase 2 config files that have enrolled fans but no Phase 3 control-profile fields (target_temp_millidegrees, aggregation, pid_gains, cadence, deadband_millidegrees, actuator_policy, pid_limits) fail to deserialize because AppliedFanEntry declares those fields as required. When load() fails, the daemon silently falls back to AppConfig::default(), dropping all enrolled fans to unmanaged — a safety regression. This plan adds serde(default) annotations so Phase 2 configs deserialize successfully with safe defaults.

Output: Two atomic commits — one fixing serde backward compat with a test, one suppressing dead_code warnings.
</objective>
