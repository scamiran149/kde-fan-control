### Test 07-01: Clean build from scratch

1. Remove the build directory: `rm -rf gui/build`
2. Configure: `cmake -S gui -B gui/build`
3. Build: `cmake --build gui/build`
4. Verify: `gui_app` binary exists at `gui/build/gui_app`, no linker errors, no compiler warnings

**Pass criteria:** Build completes with exit code 0. Binary exists and is executable.

### Test 07-02: Launch without daemon (disconnected state)

1. Ensure the fan-control daemon is NOT running (check with `systemctl status kde-fan-control` or `busctl --system list | grep FanControl`)
2. Start the GUI: `gui/build/gui_app`
3. Verify:
   - Window opens with Kirigami ApplicationWindow shell
   - Overview page shows "Daemon disconnected" severity banner (Error type InlineMessage)
   - System tray icon appears with "network-offline-symbolic" icon
   - No crashes or assertion failures in console output
   - No undefined symbol errors related to KNotification or KStatusNotifierItem

**Pass criteria:** Application launches and displays disconnected state. Tray icon shows offline status. No crashes.

### Test 07-03: Launch with daemon (connected state)

1. Start the fan-control daemon: `sudo systemctl start kde-fan-control` or run the daemon directly
2. Start the GUI: `gui/build/gui_app`
3. Verify:
   - Window opens and Overview page populates with fan/sensor data within 3-5 seconds
   - System tray icon changes to reflect severity (emblem-ok-symbolic for managed, dialog-information-symbolic for unmanaged)
   - Fan rows show state badges, temperature values, output bars
   - Clicking a fan row navigates to FanDetailPage
   - Fan detail page shows core controls (enrollment toggle, control mode, sensor sources)
   - No console errors related to DBus signal connections

**Pass criteria:** GUI displays live data from daemon. Navigation works. Tray icon reflects state.

### Test 07-04: Navigation smoke test

With GUI running and daemon connected:

1. Click "Inventory" in GlobalDrawer -> InventoryPage loads with sensor and fan lists
2. Click a fan row in Overview -> FanDetailPage loads for that fan
3. Press back (toolbar or keyboard) -> returns to Overview
4. Click "About" in GlobalDrawer -> About page appears (if implemented)
5. Verify all pages load without QML binding errors or console warnings

**Pass criteria:** All pages load correctly. No QML errors. Back navigation works.
