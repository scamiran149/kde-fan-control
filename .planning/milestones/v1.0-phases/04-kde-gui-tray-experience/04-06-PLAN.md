### Task 06-02: Implement connectDBusSignals() in StatusMonitor

**Action:** Replace the empty `connectDBusSignals()` body in `gui/src/status_monitor.cpp` with SLOT-based DBus signal connections. Since `QDBusConnection::connect()` in Qt6 requires SLOT macros (not lambdas), the relay slots on DaemonInterface are the targets:

```cpp
void StatusMonitor::connectDBusSignals()
{
    QDBusConnection bus = QDBusConnection::systemBus();

    using namespace Qt::StringLiterals;

    // Lifecycle signals at /org/kde/FanControl/Lifecycle
    bus.connect(u"org.kde.FanControl"_s,
                u"/org/kde/FanControl/Lifecycle"_s,
                u"org.kde.FanControl.Lifecycle"_s,
                u"draft_changed"_s,
                m_daemon, SLOT(onDraftChanged()));

    bus.connect(u"org.kde.FanControl"_s,
                u"/org/kde/FanControl/Lifecycle"_s,
                u"org.kde.FanControl.Lifecycle"_s,
                u"applied_config_changed"_s,
                m_daemon, SLOT(onAppliedConfigChanged()));

    bus.connect(u"org.kde.FanControl"_s,
                u"/org/kde/FanControl/Lifecycle"_s,
                u"org.kde.FanControl.Lifecycle"_s,
                u"degraded_state_changed"_s,
                m_daemon, SLOT(onDegradedStateChanged()));

    bus.connect(u"org.kde.FanControl"_s,
                u"/org/kde/FanControl/Lifecycle"_s,
                u"org.kde.FanControl.Lifecycle"_s,
                u"lifecycle_event_appended"_s,
                m_daemon, SLOT(onLifecycleEventAppended(QString,QString)));

    // Control signals at /org/kde/FanControl/Control
    bus.connect(u"org.kde.FanControl"_s,
                u"/org/kde/FanControl/Control"_s,
                u"org.kde.FanControl.Control"_s,
                u"AutoTuneCompleted"_s,
                m_daemon, SLOT(onAutoTuneCompleted(QString)));
}
```

Note: The `AutoTuneCompleted` signal name uses PascalCase on the wire (matching the daemon's `#[zbus(signal, name = "AutoTuneCompleted")]` attribute).

**Verify:** `cmake --build gui/build` compiles and links without errors.

**Action:**

   - Signal: `void pollingEnabledChanged();`
   - Private members: `QTimer *m_refreshTimer;` and `bool m_pollingEnabled = true;`
   - Include: `#include <QTimer>`

2. In `gui/src/status_monitor.cpp`:
   - In the constructor, after existing connects, create and configure the timer:
   ```cpp
   m_refreshTimer = new QTimer(this);
   m_refreshTimer->setInterval(3000); // 3 seconds
   m_refreshTimer->setSingleShot(false);
   connect(m_refreshTimer, &QTimer::timeout, this, &StatusMonitor::refreshAll);
   // Timer starts when daemon connects (see onDaemonConnectedChanged)
   ```
   - In `onDaemonConnectedChanged()`, add timer start/stop logic:
   ```cpp
   if (connected) {
       refreshAll();
       if (m_pollingEnabled) {
           m_refreshTimer->start();
       }
   } else {
       m_refreshTimer->stop();
   }
   ```
   - Implement `pollingEnabled()` and `setPollingEnabled()`:
   ```cpp
   bool StatusMonitor::pollingEnabled() const { return m_pollingEnabled; }

   void StatusMonitor::setPollingEnabled(bool enabled) {
       if (m_pollingEnabled == enabled) return;
       m_pollingEnabled = enabled;
       Q_EMIT pollingEnabledChanged();
       if (enabled && m_daemonConnected) {
           m_refreshTimer->start();
       } else {
           m_refreshTimer->stop();
       }
   }
   ```
   - In `onDaemonDisconnected()`, add `m_refreshTimer->stop();`

### Task 06-04: Connect polling pause to window visibility from QML

**Action:** In `gui/qml/Main.qml`, add a handler that pauses polling when the window is minimized or hidden:

```qml
// In the ApplicationWindow, add to existing property bindings or signal handlers:
onVisibilityChanged: {
    if (visibility === Window.Hidden || visibility === Window.Minimized) {
        statusMonitor.pollingEnabled = false
    } else {
        statusMonitor.pollingEnabled = true
    }
}
```

This ensures the 3-second polling timer pauses when the user closes to tray or minimizes, reducing unnecessary DBus traffic. Timer resumes when the window becomes visible again.
