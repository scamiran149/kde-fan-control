Create the `Kirigami.Dialog`-based wizard per UI-SPEC Wizard Contract.

**WizardDialog.qml** — `Kirigami.Dialog` with multi-step StackLayout:

**Entry points per UI-SPEC:**
- Overview empty state CTA (when no fans are managed)
- Secondary action in Overview toolbar ("Wizard configuration" per D-04)
- Secondary action in Fan Detail when fan is unmanaged

**Steps (per UI-SPEC Wizard Contract):**

1. **Select Fan** (`Controls.ComboBox` or `ListView`):
   - Model: FanListModel filtered to `supportState === "available"` AND `enrolled === false`
   - Shows: fan display name, available control modes
   - Per D-04: "optional guided setup path" — not mandatory
   - On selection: call `draftModel.loadFan(selectedFanId)`, then `draftModel.setEnrolled(true)`

2. **Select Control Mode** (`Controls.ComboBox`):
   - Model: fan's available control modes (e.g., ["PWM"] or ["PWM", "Voltage"])
   - Per D-03: control mode selector
   - On selection: call `draftModel.setControlMode(selectedMode)`

3. **Select Sensor Source(s)** (`ListView` with `Controls.CheckBox` multi-select):
   - Model: SensorListModel
   - Per D-03: multi-select sensor list
   - If exactly 1 selected: show "Single sensor" text (per UI-SPEC source selection rules)
   - If 2+ selected: show aggregation dropdown in next step
   - On completion: call `draftModel.setSensorIds(selectedIds)`

4. **Choose Aggregation** (conditional, only when 2+ sensors selected):
   - `Controls.ComboBox` with model ["Average", "Max", "Min", "Median"]
   - On selection: call `draftModel.setAggregation(selectedAggregation)`
   - If only 1 sensor selected: skip this step automatically

5. **Set Target Temperature** (`Controls.SpinBox`):
   - Label: "Target temperature"
   - Suffix: "°C"
   - Decimals: 1
   - Range: 20°C to 100°C (reasonable fan control range)
   - Default: 65°C (matching daemon default of 65000 millidegrees)
   - On change: call `draftModel.setTargetTempCelsius(value)`

6. **Review Starter PID Values**:
   - Show Kp, Ki, Kd in read-only labels with default values
   - Values come from `draftModel.kp`, `draftModel.ki`, `draftModel.kd`
   - Include brief description of each gain per UI-SPEC PID help text
   - Per D-14: "Advanced controls are not shown in the wizard"

7. **Review + Validate + Apply**:
   - Summary of all selections: fan name, control mode, sensor source(s), aggregation, target temp, PID values
   - Two actions at bottom:
     a. "Validate draft" — calls `draftModel.validateDraft()`
        - If validation fails: show per-fan error messages
        - If validation succeeds: show "Valid: {N} enrollable entries" success message
     b. "Apply changes" — calls `draftModel.applyDraft()`
        - Per UI-SPEC: wizard NEVER bypasses validation
        - Per UI-SPEC: must end on review screen with explicit apply
   - "Back" button returns to previous step
   - "Cancel" at any point: calls `draftModel.discardDraft()` and closes wizard

**Navigation:**
- Use `StackLayout` with `currentIndex` for step tracking
- "Next" and "Back" buttons in Kirigami.Dialog footer
- "Finish" button on step 7 (replaces "Next")
- Progress indicator showing current step number

**Rules per UI-SPEC:**
- Wizard uses the same backend draft/apply contract as direct editing (calls DraftModel methods)
- Wizard NEVER bypasses validation (step 7 always validates before apply)
- Wizard MUST end on a review screen with explicit apply (step 7)
- Advanced controls are NOT shown in the wizard (no cadence, deadband, actuator policy, PID limits)
- If wizard is cancelled at any point, call `draftModel.discardDraft()` to clean up

**Integration with Main.qml:**
- In Main.qml, add the `WizardDialog` component and expose it as a context property or direct QML reference
- Connect the "Wizard configuration" toolbar action and empty state CTA to `wizardDialog.open()`
- When a specific unmanaged fan is selected from Detail page, pre-select it in wizard step 1
