<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| daemon runtime → privileged sysfs writes | A failure-path handler writes directly to hardware control nodes |
| daemon process lifetime → persisted incident state | Safety evidence must survive restarts without being forged by clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | D | daemon failure path | mitigate | Centralize fallback handling so owned-fan safe-maximum writes run from both graceful shutdown and in-process unexpected-exit hooks |
| T-02-06-02 | T | persisted fallback incident state | mitigate | Persist fallback incidents only through daemon-owned writes, keyed from `OwnedFanSet`, and reload them before rebuilding runtime state |
| T-02-06-03 | I | runtime status / lifecycle events | mitigate | Keep fallback incidents distinct from degraded boot mismatch events and expose affected fan IDs plus timestamps after restart |
| T-02-06-04 | E | unmanaged fan safety boundary | mitigate | Record and write fallback only for fans already claimed in `OwnedFanSet`; never derive fallback targets from draft/applied config alone |
</threat_model>
