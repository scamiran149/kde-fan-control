Create the C++ tray icon and notification handler.

**1. TrayIcon** (`gui/src/tray_icon.h` and `.cpp`):
- Class `TrayIcon : public QObject` with `QML_ELEMENT`
- Uses `KStatusNotifierItem` from KF6 (not QSystemTrayIcon)
- Constructor: Create `KStatusNotifierItem` with `this` as parent, set category to `SystemServices`
- Q_PROPERTY: `QString worstSeverity` â€” computed from fan list (fallback > degraded > high-temp > managed > unmanaged > disconnected per UI-SPEC precedence)
- Q_PROPERTY: `int managedFanCount` â€” count of fans with state "managed"
- Q_PROPERTY: `int alertCount` â€” count of fans with state "degraded" or "fallback" or highTempAlert
- Q_PROPERTY: `bool daemonConnected` â€” forwarded from StatusMonitor
- Q_PROPERTY: `bool hasStickyAlerts` â€” true when alertCount > 0

**Icon management per UI-SPEC severity precedence:**
- fallback â†’ icon: `dialog-error-symbolic`, status: `NeedsAttention`
- degraded â†’ icon: `data-warning-symbolic`, status: `NeedsAttention`
- high-temp alert â†’ icon: `temperature-high-symbolic`, status: `NeedsAttention`
- managed healthy â†’ icon: `emblem-ok-symbolic`, status: `Active`
- no managed fans / unmanaged only â†’ icon: `dialog-information-symbolic`, status: `Passive`
- daemon unavailable â†’ icon: `network-offline-symbolic`, status: `Passive`

**Tooltip** per tray precedence:
- Primary line: severity summary (e.g., "1 fan degraded", "All fans managed", "Daemon disconnected")
- Secondary line: "{managedFanCount} managed, {alertCount} alerts"

**ContextMenu actions:**
- "Open Fan Control" â†’ activates/shows the main window
- "Acknowledge alerts" â†’ only when `hasStickyAlerts === true`; sets an acknowledged flag to clear stickiness per D-12
- "Quit" â†’ closes the application

**Popover association:**
- Set `KStatusNotifierItem` to provide a custom QML popover â€” `TrayPopover.qml`
- Use `KStatusNotifierItem::setAssociatedWidget()` to point to the main application window

**Connecting to models:**
- Connect StatusMonitor's `control_status_changed` and `degraded_state_changed` to trigger `updateSeverity()` which iterates FanListModel to compute worstSeverity, managedFanCount, alertCount
- Connect StatusMonitor's `daemonConnected` property to update `daemonConnected` and tray icon state
- On `updateSeverity()`, call `KStatusNotifierItem::setIconByName()`, `setToolTip()`, `setStatus()` to reflect current state

**2. NotificationHandler** (`gui/src/notification_handler.h` and `.cpp`):
- Class `NotificationHandler : public QObject`
- Uses `KNotification` from KF6 for desktop notifications
- Constructor: Connect to StatusMonitor's DBus signal subscriptions

**Notification trigger conditions per D-11 (only on TRANSITION into):**
- `degraded_state_changed` signal: Check if any fan transitioned INTO degraded state. If so, emit notification:
  - Title: "Fan control degraded"
  - Text: "One or more managed fans could not be controlled safely. Open Fan Control to review the reason."
  - Priority: High
- Fallback transition: If any fan enters fallback state, emit:
  - Title: "Fallback active"
  - Text: "Managed fans were driven to safe maximum output. Open Fan Control now."
  - Priority: High (more urgent than degraded)
- High-temp alert: If `control_status_changed` shows `high_temp_alert` transition to true for a fan:
  - Title: "High temperature alert"
  - Text: "A managed fan is above its target temperature. Check runtime status and tuning."
  - Priority: Normal

**Stickiness per D-12:**
- Desktop popup may be transient (KNotification default behavior)
- Alert must remain sticky in tray popover (TrayPopover shows active alert summary) AND in main window banner (OverviewPage banners) until acknowledged
- "Acknowledge alerts" in tray context menu clears UI stickiness only per UI-SPEC: "Acknowledging clears UI stickiness only; it does not alter daemon state"
- NotificationHandler tracks acknowledged state; once acknowledged, clears the sticky flag in TrayIcon

**3. Update main.cpp** from Plan 01:

- Register `TrayIcon` as QML context property `trayIcon`
- Pass `statusMonitor` reference to `TrayIcon` and `NotificationHandler` constructors
- Ensure QApplication (not QGuiApplication) is used for KStatusNotifierItem compatibility
- Add KF6::Notifications and KF6::StatusNotifierItem to CMake target link libraries
