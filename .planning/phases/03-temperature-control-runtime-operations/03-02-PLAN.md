<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Live sensor sysfs → daemon control loop | Kernel-reported temperatures are untrusted runtime inputs that affect privileged writes. |
| Daemon control loop → pwm sysfs | Privileged actuator writes can affect thermal safety if issued to the wrong fan or with wrong bounds. |
| DBus read client → control status | Unprivileged callers may inspect runtime data but must not gain write capability. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | live sensor sampling | mitigate | Treat unreadable or missing sensors as degraded runtime state and stop writes for that fan instead of reusing stale values. |
| T-03-05 | Elevation of privilege | pwm write path | mitigate | Gate writes by `OwnedFanSet`, keep mapping within configured PWM bounds, and preserve Phase 2 root-only daemon ownership. |
| T-03-06 | Information disclosure | `get_control_status` | accept | Runtime temperatures and control state are intentionally read-open per product policy; no secrets are exposed. |
</threat_model>
