| Boundary | Description |
|----------|-------------|
| CLI → DBus control methods | CLI sends user-entered values to the daemon but must not become a second authority surface. |
| CLI text output → operator decisions | Status and tune output must clearly distinguish staged versus live values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Spoofing | CLI privilege expectations | accept | CLI relies on daemon-side authorization and reuses existing root-required error messaging rather than inventing client-side auth. |
| T-03-11 | Tampering | CLI profile payload construction | mitigate | Serialize the exact daemon contract fields into JSON and keep conversions explicit (`degC` → `target_temp_millidegrees`). |
| T-03-12 | Repudiation | staged-vs-live operator flow | mitigate | `auto-tune accept` and `control set` must state that values are staged only and require `apply` to become live. |
</threat_model>
