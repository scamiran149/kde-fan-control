<task type="auto">
  <name>Task 1: Create control domain module with PID controller, aggregation, and auto-tune types</name>
  <files>crates/core/src/control.rs, crates/core/src/lib.rs</files>
  <action>
Create `crates/core/src/control.rs` with:
1. `AggregationFn` enum with variants `Average`, `Max`, `Min`, `Median`. Each variant's `compute(readings: &[f64]) -> f64` method handles empty/single-element inputs gracefully (empty returns 0.0, single returns the value).
2. `PidGains` struct with fields `kp: f64`, `ki: f64`, `kd: f64`.
3. `PidController` struct holding `gains: PidGains`, `target_millidegrees: i64`, `integral: f64`, `previous_error: Option<f64>`, `tick_interval_ms: u64`, `output_min: f64` (default 0.0), `output_max: f64` (default 255.0). Method `compute(&mut self, current_millidegrees: i64) -> PidOutput`:
   - Convert millidegrees to degrees for error computation
   - error = target - current (in degrees)
   - P term = Kp * error
   - I term = Ki * (integral + error * dt_seconds), with anti-windup: clamp integral to ±max_integral (derive max from output_max / Ki, or use 1000.0 as default bound)
   - D term = Kd * (error - previous_error) / dt_seconds (first tick: D = 0)
   - output = P + I + D, clamped to [output_min, output_max]
   - Return PidOutput with output, p_term, i_term, d_term, error, integral
   - Reset method `reset(&mut self)` to clear integral and previous error
4. `PidOutput` struct with fields `output: f64`, `p_term: f64`, `i_term: f64`, `d_term: f64`, `error: f64`, `integral: f64`, `current_millidegrees: i64`, `aggregated_millidegrees: i64`.
5. `AutoTuneResult` struct with `proposed_gains: PidGains`, `lag_time_ms: u64`, `max_rate: f64`, `step_duration_ms: u64`.
6. `AutoTuneState` enum: `Idle`, `Running { fan_id: String, start_time: std::time::Instant }`, `Completed { fan_id: String, result: AutoTuneResult }`.

Ensure AggregationFn derives `Serialize, Deserialize, Clone, Copy, Debug, PartialEq, Eq` with `#[serde(rename_all = "snake_case")]`.
Ensure PidGains derives `Serialize, Deserialize, Clone, Copy, Debug, PartialEq]`.
Add `pub mod control;` to `crates/core/src/lib.rs`.

Include comprehensive unit tests:
- Aggregation: average of [45.0, 47.0, 50.0] = 47.333..., max = 50.0, min = 45.0, median = 47.0
- Aggregation: single-element and empty-input edge cases
- PidOutput: proportional-only (Ki=0, Kd=0) reduces error toward target
- PidController: integral accumulates over ticks toward setpoint
- PidController: anti-windup clamps integral when output saturates
- PidController: derivative term responds to rate of change
- PidController: output is clamped to [0, 255]
- PidController: reset clears internal state
  </action>
  <verify>
    <automated>cargo test -p kde-fan-control-core control -- --nocapture</automated>
  </verify>
  <done>PID controller computes correct output with P/I/D terms, anti-windup, and clamping. Aggregation functions handle all four modes plus edge cases. Types are serializable and tested.</done>
</task>
