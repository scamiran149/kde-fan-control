<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Draft config → applied control profile | User-supplied PID values and actuator settings cross into daemon-owned safety logic. |
| PID runtime contract → hardware writer | Logical output is later mapped into privileged sysfs writes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `crates/core/src/config.rs` | mitigate | Reject missing targets, empty sensor groups, invalid cadences, and invalid actuator bounds before `apply_draft` can promote them. |
| T-03-02 | Denial of service | `crates/core/src/control.rs` | mitigate | Clamp integral, derivative, and final output; keep deadband and logical output bounds explicit so unstable inputs cannot run away. |
| T-03-03 | Elevation of privilege | `crates/core/src/lifecycle.rs` runtime payloads | accept | Runtime payloads remain read-only data contracts; no new privileged write path is introduced in this plan. |
</threat_model>
